Icons consistency

REDESIGN UX UI B2B DESIGN SYSTEM MATERIAL DESIGN

I designed and standardized the entire icon system for the Hubject portal, collaborating closely with developers to achieve a new icon-font solution. This new icon system significantly improved consistency, accessibility, and reduced delivery time by 50%.

Hubject's product offers the largest EV-charging network, helping B2B players: CPOs (charging point operators) and EMPs (electric mobility providers) build lasting connections for a smooth charging experience for EV drivers.


My role: Product designer (end-to-end UX/UI design to hand-over and testing)
Project duration: 3 Months
Team: 1 Product designers, 1 support UX designer, 5 FE developers

The Hubject portal design system is built with Angular front-end architecture and follows Material Design guidelines. The design system component library is built in Figma, with a base file for core styles and another for components tailored to the platform's B2B needs.


üë∑ The UI/Front-End consistency initiative

When I joined Hubject, key components weren't aligned between Figma and front-end code.
To address this, I initiated a UX/UI consistency effort in collaboration with front-end developers to standardize core components. Icons quickly emerged as one of the biggest pain points‚Äîboth visually and in terms of implementation.

The "Available contracts" page on Hubject portal

üö© The problem

Icons came from various sources, leading to visual inconsistency and a confusing user experience. The lack of a unified style disrupted users' mental models and weakened brand perception.
Additionally, relying on SVGs caused interaction issues and slowed down development.


üéØ Design goal

The main goal was to create a consistent and scalable icon system that would be easy for developers to implement. Each icon needed to be clear in meaning, follow a unified visual style, and be quick to apply in code.
It was also important to make sure the style matched the brand's tone‚Äîclean, reliable, and friendl.

üöÄ The solution

The new partner finder and profile page accelerate B2B client's network expantion task by bringing research tools directly into the platform.

The flow starts with a list of all B2B sellers. Clicking on a partner item leads to a detailed profile with network data, contracts, and contact info.

Display of the new B2B seller finder

The multi-selection filter was chosen for country coverage, enabling users to find B2B sellers that meet their specific country combination needs

The min-max filter was chosen for the number of charging points to accommodate the quantitative nature of the criteria, allowing users to filter based on their specific network size requirements.

üìà The outcome

I delivered a unified set of 190+ icons, implemented across 8 applications and 30 shared components. The new system improved consistency, accessibility, and user experience, while reducing developer implementation time by 50%. It also received positive feedback from users and stakeholders.


üå± Key learning

A major learning came during implementation - rolling out new design system elements takes time and adaptability. Breaking the project into smaller, manageable steps helped maintain momentum and made the work feel more achievable.
